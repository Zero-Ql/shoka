# 物品 (Item)

物品【Item】中存储了一些不会变化的信息，比如堆叠上限，最大耐久等。

所有此种物品【Item】的物品堆【ItemStack】都以该物品【Item】实例为基础，比如你的物品栏有三个泥土物品堆【ItemStack】，数量分别为 10 ， 20 ， 30 ，通过这三个物品堆【ItemStack】获取到的都是同一个物品【Item】实例，即泥土物品【Item】实例。(物品【Item】的一些配置信息将会影响到物品堆【ItemStack】与物品实体的效果)

不要把可变的信息存储到物品【Item】中！请把它们存储到物品堆【ItemStack】里！

物品堆【ItemStack】中存储了一些额外的，可变的信息，比如物品【Item】的数量，药水中的效果种类等。它们以 nbt 的形式存储在物品堆【ItemStack】中。

一个物品堆【ItemStack】拥有下面的三个主要部分：

  • 它代表的物品【Item】，可以使用 itemstack.getItem() 获得。
  • 物品【Item】的数量，一般在 1~64 之间，通过 itemstack.getCount() 获得，使用 itemstack.setCount(int) 或 itemstack.shrink(int) 变化。
  • 额外的 nbt 数据，存储着每个物品堆【ItemStack】独特的数据。通过 itemstack.getTag() 获取，或者通过 itemstack.getOrCreateTag() 来处理标签不存在的情况。 ItemStack 类中也存在许多其他和 nbt 有关的方法，最重要的是 hasTag() 和 setTag() 。

物品堆【ItemStack】非单例，即两个数据完全相同的物品堆【ItemStack】并不会共用同一个实例 (对象)

值得一提的是，拥有空白的额外 nbt 的物品堆【ItemStack】和不拥有 nbt 的物品堆【ItemStack】是不同的。这意味着他们无法被堆叠，尽管他们在功能上完全一样。如果需要创建一个新的物品堆【ItemStack】，请使用 new ItemStack(Item) 并传入指定的物品【Item】。默认情况下，这样创建的物品堆【ItemStack】数量为 1 且没有 nbt 数据；但同时，类中也提供了创建带有数量或 nbt 物品的构造方法。

# 注册

注册是指将模组中的对象（如物品、方块、声音等）告知游戏的过程。注册非常重要，因为如果没有注册，游戏将不知道这些对象的存在，这会导致不可预测的行为和崩溃。
游戏中大多数需要注册的对象都是通过 Forge 注册表来处理的。注册表类似于一个映射表，它将值分配给键。 Forge 使用带有 ResourceLocation 键的注册表来注册对象。这使得 ResourceLocation 可以作为对象的 “注册名称”。
每种可注册的对象类型都有自己的注册表。要查看所有由 Forge 封装的注册表，可以查看 ForgeRegistries 类。注册表内的所有注册名称必须是唯一的。然而，不同注册表中的名称不会冲突。例如，有一个方块注册表和一个物品注册表。一个方块和一个物品可以使用相同的名称 example:thing 进行注册而不会冲突；但是，如果两个不同的方块或物品使用完全相同的名称进行注册，后注册的对象将覆盖先注册的对象。

以上信息来自官方文档

原文

Registration is the process of taking the objects of a mod (such as items, blocks, sounds, etc.) and making them known to the game. Registering things is important, as without registration the game will simply not know about these objects, which will cause unexplainable behaviors and crashes.

Most things that require registration in the game are handled by the Forge registries. A registry is an object similar to a map that assigns values to keys. Forge uses registries with ResourceLocation keys to register objects. This allows the ResourceLocation to act as the “registry name” for objects.

Every type of registrable object has its own registry. To see all registries wrapped by Forge, see the ForgeRegistries class. All registry names within a registry must be unique. However, names in different registries will not collide. For example, there’s a Block registry, and an Item registry. A Block and an Item may be registered with the same name example:thing without colliding; however, if two different Blocks or Items were registered with the same exact name, the second object will override the first.

——from Neoforge docs , https://docs.neoforged.net/docs/concepts/registries

# 物品特性 (ItemProperties)

物品特性用于描述物品在特定情况下表现出的特征，比如是否可以食用，是否抗火，最大耐久与稀有度等。

所有可以用 Properties 做到的方法都在 net.minecraft.world.item.Properties 类中。
