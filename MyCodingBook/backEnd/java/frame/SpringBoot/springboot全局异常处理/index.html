# SpringBoot 全局异常处理

​ 在写程序时总是需要处理某些异常，虽然 Java 提供了自带的异常处理，但是一个完整的项目中可能遇到千奇百怪的异常，这些异常如果都需要我们手动去进行处理，那么代码量会增多不说，程序之间的耦合还会增大不少，这并不符合 “高内聚，低耦合” 的理念。

​ Spring 框架提供了 @RestControllerAdvice 注解来处理全局异常，这样可以避免大量的 try catch ，也方便了异常信息的统一处理和维护。

注：注解的作用请见：spring 开发注解集合 | 渡世之舟 (zero-li.fun)
