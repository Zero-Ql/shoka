# Docker 使用

  • 在 docker 容器内运行应用程序

docker run ubuntu:15.10 /bin/echo "hello world"

参数解析：

  1. docker :Docker 的二进制执行文件
  2. run : 与 Docker 组合运行一个容器
  3. ubuntu:15.10 : 要运行的镜像，本地没有就会从镜像仓库下载
  4. /bin/echo "hello world" : 在启动的容器里执行 echo 命令
  • 与 docker 容器进行交互

docker run -t -i centos:centos7 /bin/bash

参数解析：

  1. -t : 在新的容器里指定一个终端
  2. -i : 允许与指定的终端进行交互

注意：可以用 exit 退出伪终端

  • 通过 ps 查看正在运行的容器

docker ps

参数解析：

  1. CONTAINER ID : 容器 ID

  2. IMAGE : 使用的镜像

  3. COMMAND : 启动镜像时运行的命令

  4. CREATED : 容器的创建时间

  5. STATUS : 容器的状态 （容器的七种状态：

    1. created 已创建

    2. restaring 重启中

    3. running 或 Up 运行中

    4. removing 迁移中

    5. paused 暂停

    6. exited 停止

    7. dead 死亡）

  6. PORTS : 容器的端口信息和使用的连接类型（ TCP/UDP ）

  7. NAMES : 自动分配的容器名称

# 常用命令
  1. docker images 列出本地主机上的镜像
  2. docker search centos[:7] 搜索镜像
  3. docker pull centos[:centos7] 下载镜像
  4. docker tag centos:centos7 mycentos:centos7 为镜像添加标签，其中分号前为镜像类型，分号后为镜像版本
  5. docker history centos:centos7 查看镜像的构建历史信息
  6. docker rmi centos:centos7 删除镜像（可以使用 -f 强制删除，在删除使用 tag 标签标记的镜像时可以不用加 -f 参数）
  7. docker image prune [-a] [-f] 清理镜像，加 -a 参数即删除所有无用的镜像，不仅仅是临时镜像 -f 参数即可强制删除镜像而不提示
  8. docker image save centos:centos7 > ./my-centos7.tar.gz 导出镜像到当前目录，格式为一个压缩包（可以自行选择导出的位置）
  9. docker image load [-i] [-q] ./my-centos7.tar.gz 将 centos7 的镜像压缩包导入， -i 参数是指定导入的文件， -q 参数是精简输出信息
  10. docker image inspect [-f] { {".Os"} } centos:centos7 查看镜像信息，可以使用 -f 参数获取指定信息
  11. docker restart $(docker ps -a -q) 重启所有 docker 容器

注意：[ ] 里的为可选内容，选择具体版本
